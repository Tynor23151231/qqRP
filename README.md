# qqRP Bot

Telegram Business Bot на **aiogram 3.x** для RP-действий через точку (`.муа`, `.обнять`, `.цветы` и т.д.) — работает от имени владельца аккаунта через **Telegram Business Connection**.

## Возможности

- Dot-команды (`.муа`, `.обнять`, `.прижать`, `.цветы`, `.кольцо`, `.подарок`, `.сердце`, `.погладить`, `.печенье`, `.кофе`, `.торт`, `.дай5`, `.шлеп`, `.пнуть`, `.удар`, `.тык`, `.буп`, `.укус`, `.кусь`, `.снежок`, `.бан`, `.лизь`), автоматически определяющие цель:
  - reply на сообщение → автор этого сообщения;
  - `@username`;
  - в личном чате без reply/username → собеседник по умолчанию.
- Работают и в **Business-чатах** (от лица владельца через Telegram Business Connection), и в обычных **группах/супергруппах**.
- **Премиум-эмодзи, зависящие от пола участников.** Для `.муа`/`.обнять`/`.прижать` набор эмодзи подбирается по паре (М→Ж, Ж→М, Ж→Ж, М→М); для `.сердце`/`.погладить`/`.кофе`/`.торт`/`.дай5`/`.шлеп`/`.кусь` — по полу автора; остальные действия используют фиксированный набор для всех. Если пол цели ещё не известен боту, используется дефолт по полу автора.
- Ключевые слова-варианты: `.лизь попу` / `.лизь попа` подставляют отдельный эмодзи вместо стандартного набора.
- Упоминания — кликабельные ссылки на профиль (`text_link`: `https://t.me/username` или `tg://user?id=`, если username нет) с отключённым link preview, чтобы не было превью-карточек.
- Кастомное отображаемое имя (`custom_name`) — если задано, используется вместо имени из Telegram.
- Склонение по полу (`/start` → выбор пола → `Иван поцеловал` / `Анна поцеловала`).
- **Пользовательские RP-действия (премиум):** мастер `/addrp` в личном чате с ботом — триггер → один или несколько премиум-эмодзи одним сообщением (при использовании один выбирается случайно, как у встроенных действий) → шаблон текста.
- `/profile`, `/stats`, `/settings` (случайные анимации, компактный режим, случайные шаблоны, язык).
- Алиасы команд (например `.поцеловать` = `.муа`) через `"aliases"` в actions.json.
- **`.typing <текст>`** — постепенное "печатание" сообщения по нарастающей (с, со, соо, ...), как будто его набирает человек. Платная функция (см. ниже).
- **Премиум-подписка за Telegram Stars.** `/premium` — статус и покупка (99 ⭐️ / 30 дней по умолчанию, настраивается в `.env`). Открывает `.typing`. Повторная покупка продлевает срок, а не сбрасывает его.
- **Админ-консоль.** Telegram ID из `ADMIN_IDS` в `.env` получают доступ к `/admin`, `/grant_premium`, `/revoke_premium`, `/premium_list`, `/find_user` — выдача/отзыв премиума по `@username` или числовому ID, без доступа к базе напрямую.
- Удаление исходной команды пользователя после обработки (если есть права — в Business через `delete_business_messages`, в группах через обычный `delete_message`).
- Полностью асинхронный стек: aiogram 3.29.x + SQLAlchemy Async + PostgreSQL.

## Структура проекта

```
app/
  handlers/
    business/          # business_message и business_connection (Business-чаты)
    group_actions.py    # dot-команды в обычных группах/супергруппах
    start.py, profile.py, stats.py, settings.py, custom_rp.py
  database/             # engine, session maker, light migrations
  models/                # User, ActionLog, CustomTrigger
  middlewares/            # DatabaseMiddleware, UserMiddleware
  services/               # ActionService (рендер действий), UserService
  utils/                  # парсинг команд, построение MessageEntity, UTF-16 офсеты
  keyboards/              # inline-клавиатуры
  data/actions.json       # встроенные действия — расширяются без изменения кода
  config.py
main.py
```

## Запуск

```bash
cp .env.example .env       # заполнить BOT_TOKEN и DATABASE_URL
pip install -r requirements.txt
python main.py
```

Бот сам создаст таблицы в PostgreSQL при первом запуске (`init_models()`) и подтянет
недостающие колонки через лёгкие идемпотентные `ALTER TABLE ... ADD COLUMN IF NOT EXISTS`
(см. `app/database/base.py`). Для более серьёзных изменений схемы рекомендуется завести Alembic.

## Премиум и админ-консоль

Настройки в `.env`:

```bash
ADMIN_IDS=123456789,987654321   # Telegram ID через запятую — свой узнать у @userinfobot
PREMIUM_PRICE_STARS=99          # стоимость подписки в Telegram Stars
PREMIUM_DURATION_DAYS=30        # длительность подписки в днях
```

- **Покупка (для любого пользователя):** `/premium` в личном чате с ботом → кнопка "Купить премиум" → оплата Telegram Stars прямо в Telegram (без банковских карт). После оплаты `premium_until` сохраняется в БД (см. `app/models/user.py`, `has_premium`) и открывает платные функции. Повторная покупка продлевает срок от текущей даты окончания, а не сбрасывает его.
- **Платные функции:**
  - `.typing <текст>` — постепенное "печатание".
  - `/addrp` — создание собственных РП-действий, в том числе с несколькими премиум-эмодзи в одном действии (случайный выбор при каждом использовании). И создание, и последующее использование уже созданных действий требуют активного премиума — если подписка закончится, действие перестанет срабатывать, пока премиум не продлят (но сама команда не удаляется, только временно недоступна).
- **Админ-консоль (только для ID из `ADMIN_IDS`):**
  - `/admin` — список доступных команд.
  - `/grant_premium @username [дней]` или `/grant_premium 123456789 [дней]` — выдать/продлить премиум (по умолчанию `PREMIUM_DURATION_DAYS`).
  - `/revoke_premium @username_или_id` — снять премиум досрочно.
  - `/premium_list` — все пользователи с активной подпиской и датой окончания.
  - `/find_user @username_или_id` — быстрая карточка пользователя (пол, премиум, статистика).
  - Поиск по числовому ID работает только если пользователь уже хоть раз писал боту — это ограничение Telegram Bot API (`get_chat` не резолвит произвольные ID), а не бага.

## Деплой на Railway

В репозитории уже есть всё нужное:

- Автоопределение Nixpacks по `requirements.txt` (Python + venv + pip настраиваются автоматически — свой `nixpacks.toml` тут не нужен и может всё сломать, если переопределить `nixPkgs` без `pip`).
- `Procfile` — `worker: python main.py` (это воркер, не веб-сервис — публичный домен не нужен).
- `railway.json` — билдер NIXPACKS, `startCommand`, restart policy `ON_FAILURE`.
- `.python-version` — подсказка Nixpacks по версии Python (3.12).

Шаги:

1. Создай проект на Railway → **Deploy from GitHub repo**.
2. Добавь плагин **PostgreSQL** — Railway сам создаст переменную `DATABASE_URL`.
   Важно: Railway отдаёт её как `postgres://` или `postgresql://` — `app/config.py` **автоматически** переписывает схему в `postgresql+asyncpg://`, ничего вручную менять не нужно.
3. В Variables добавь `BOT_TOKEN` (токен из @BotFather), а также `ADMIN_IDS` (твой Telegram ID и, если нужно, других админов через запятую) — без него `/grant_premium` и остальная админ-консоль будут недоступны никому. `PREMIUM_PRICE_STARS`/`PREMIUM_DURATION_DAYS` можно не задавать — по умолчанию 99 ⭐️ / 30 дней.
4. Убедись, что сервис создан как **Worker**, а не Web (Railway иногда пытается назначить `PORT` и ждать HTTP-ответ — polling-боту это не нужно; в `railway.json` явно задан `startCommand`, так что Railway не будет ждать открытия порта).
5. Deploy — при первом запуске бот сам создаст таблицы в подключённой PostgreSQL.

## Добавление нового встроенного действия

Просто добавь новый ключ в `app/data/actions.json` — код менять не нужно. Три режима подбора эмодзи:

```json
"обливание": {
  "verb": {"male": "облил", "female": "облила"},
  "template": "{user} {verb} {target} водой",
  "emoji_mode": "fixed",
  "emojis": [{"emoji": "🌊", "id": "1234567890123456789"}]
}
```

- `"emoji_mode": "fixed"` — один и тот же набор эмодзи для всех.
- `"emoji_mode": "actor_gender"` — набор зависит только от пола автора (`by_gender: {"male": [...], "female": [...]}`).
- `"emoji_mode": "pair"` — набор зависит от пары полов автор→цель (`pairs: {"male_female": [...], "female_male": [...], "female_female": [...], "male_male": [...]}`).

`"id"` — это `emoji-id` премиум-эмодзи (из `<tg-emoji emoji-id="...">`), `"emoji"` — обычный юникод-фолбэк на случай, если у получателя нет Telegram Premium. `"id": null` — просто обычный эмодзи без премиума.

Можно также добавить `"keyword_variants"` — альтернативный набор эмодзи, если после команды идёт определённое слово (например `.лизь попу` вместо стандартного `.лизь`).

## Технические детали, на которые стоит обратить внимание

- Telegram Bot API не позволяет одновременно использовать `parse_mode` и `entities` в одном сообщении, поэтому упоминания и премиум-эмодзи собираются вручную через `EntityTextBuilder` (`app/utils/entity_builder.py`) с точным подсчётом смещений в UTF-16 code units (`utf16_len`) — это важно, т.к. Python `len()` считает кодовые точки и даёт неверные офсеты для эмодзи вне BMP (например 😘, 🤗, ZWJ-последовательности типа 👨‍❤️‍💋‍👨).
- Упоминания сделаны через `text_link` (`https://t.me/username` или `tg://user?id=`), а не через `text_mention` — последний ненадёжно рендерится в Business-сообщениях.
- `link_preview_options=LinkPreviewOptions(is_disabled=True)` — чтобы ссылки-упоминания не разворачивались в превью-карточки.
- `ActionService` — единая точка входа для рендера и встроенных, и пользовательских действий (принцип открытости/закрытости: новые встроенные действия добавляются только через JSON).
